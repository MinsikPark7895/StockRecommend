# 보안 가이드 (Security Guide)

## 🔒 환경 변수 설정

이 프로젝트는 민감한 정보를 환경 변수로 관리합니다. **절대 `.env` 파일을 Git에 커밋하지 마세요.**

### 초기 설정

1. `env.example` 파일을 복사하여 `.env` 파일을 생성하세요:
   ```bash
   cp env.example .env
   ```

2. `.env` 파일을 열고 모든 값을 실제 값으로 변경하세요:
   - `POSTGRES_PASSWORD`: 강력한 비밀번호로 변경
   - `DATABASE_URL`: 비밀번호를 실제 값으로 변경
   - `POSTGRES_PORT`: 프로덕션에서는 비표준 포트 사용 권장 (선택사항)
   - `APP_PORT`: 프로덕션에서는 비표준 포트 사용 권장 (선택사항)
   - 기타 API 키 등 필요한 값 설정

### 보안 권장사항

#### 비밀번호 생성
- 최소 16자 이상 사용
- 대소문자, 숫자, 특수문자 포함
- 예측 불가능한 랜덤 문자열 사용
- 비밀번호 관리자 사용 권장

#### 프로덕션 환경
- 개발용 비밀번호와 프로덕션용 비밀번호를 다르게 설정
- 정기적으로 비밀번호 변경
- 환경별로 별도의 `.env` 파일 사용
- **포트 번호**: 기본 포트(5432, 8000) 대신 비표준 포트 사용 권장
  - 포트 스캔 공격 위험 감소
  - 하지만 포트만으로는 접근 불가 (인증이 더 중요)
  - 방화벽과 접근 제어가 포트 번호보다 더 중요

## 🚫 절대 하지 말아야 할 것

- ❌ `.env` 파일을 Git에 커밋
- ❌ 비밀번호를 코드에 하드코딩
- ❌ 기본 비밀번호를 프로덕션에서 사용
- ❌ 공개 채널(이슈, PR 등)에 비밀번호 공유

## ✅ 안전한 방법

- ✅ `.env` 파일은 `.gitignore`에 포함됨
- ✅ `env.example`은 템플릿만 포함 (실제 비밀번호 없음)
- ✅ 모든 민감한 정보는 환경 변수로 관리
- ✅ 코드 리뷰 시 하드코딩된 비밀번호 확인

## 🔍 보안 점검

프로덕션 배포 전 확인사항:

- [ ] `.env` 파일이 Git에 커밋되지 않았는지 확인
- [ ] 모든 기본 비밀번호가 변경되었는지 확인
- [ ] 강력한 비밀번호가 설정되었는지 확인
- [ ] 데이터베이스 접근이 제한되어 있는지 확인
- [ ] 포트 번호가 비표준 포트로 변경되었는지 확인 (프로덕션, 선택사항)
- [ ] 방화벽 규칙이 올바르게 설정되었는지 확인
- [ ] SSL/TLS 연결이 활성화되어 있는지 확인 (프로덕션)

## 📞 보안 문제 발견 시

보안 취약점을 발견하셨다면:
1. 즉시 이슈를 생성하지 마세요 (공개되지 않도록)
2. 프로젝트 관리자에게 직접 연락하세요
3. 책임감 있는 공개(Responsible Disclosure) 원칙을 따르세요
